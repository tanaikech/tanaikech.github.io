Privacy Policy for Workspace & Gemini AI Orchestration Engine

Last Updated: June 10, 2026

1. Introduction

This Privacy Policy outlines how the Workspace & Gemini AI Orchestration Engine (hereafter referred to as "the Application"), developed by Kanshi Tanaike, accesses, processes, and protects user data obtained via Google OAuth API scopes.

The Application is a high-performance developer utility and autonomous agent scaffolding engine designed to evaluate, benchmark, and optimize integrations between Google Workspace services, the Google Apps Script API, and Generative AI infrastructure (including Gemini API and Vertex AI endpoints via Model Context Protocol). We are profoundly committed to maintaining the absolute confidentiality, integrity, and security of your digital assets.

2. Information Collected and Scopes Accessed

The Application requests access to specific Google User Data strictly via OAuth 2.0 authentication scopes authorized explicitly by the user. Depending on the specific orchestration module executed, these scopes may include, but are not limited to:

  • Google Apps Script API Scopes: To programmatically create, modify, deploy, and benchmark Apps Script projects (e.g., .../auth/script.projects, .../auth/drive).
  • Google Workspace Service Scopes: To orchestrate cross-platform developer workflows and evaluate high-performance batch requests.
  • Gemini API / Generative AI Gateway Credentials: To facilitate Function Calling, Agent Skills execution, and Agent-to-Agent (A2A) communication protocols.

3. Strict Purpose Limitation and Use of Data

Any data accessed through Google OAuth scopes is used exclusively to fulfill the user's explicit runtime requests:

  • No Background Harvesting: Data is pulled only during active, user-initiated cryptographic sessions.
  • Contextual Execution: The Application processes your script structures, project configurations, and metadata solely to execute requested developer tasks, structural analysis, and autonomous multi-agent scaffolding simulation.
  • AI Policy Alignment: We explicitly guarantee that no user data, code snippets, or Workspace information accessed via the Google APIs is ever used to retrain, fine-tune, or otherwise improve any generative AI models or language model baselines.

4. Zero-Retention Data Storage Architecture

The Application operates on a rigorous, mathematically isolated Zero-Retention Model:

  • In-Memory Processing: All tokens, user payloads, script source codes, and API responses are handled purely in-memory (volatile RAM) or client-side within the user's active browser/execution terminal context.
  • No Persistent Logging: We do not maintain, host, or control any external database, persistent storage tier, or tracking server that captures your private technical configurations or personal identifiers.
  • Transient Token Isolation: OAuth access tokens are treated as transient assets and are never hardcoded or cached beyond the expiration dictated by Google's authentication server.

5. Third-Party Data Sharing and Commercial Non-Disclosure

We enforce an absolute prohibition on data commercialization:

  • No Third-Party Disclosures: No data accessed via Google OAuth scopes is ever shared, rented, sold, or transferred to third-party software vendors, external entities, or analytics firms.
  • No Advertising/Tracking Integration: The Application is entirely free of ad networks, behavioral tracking code, or marketing pixels.
  • End-to-End Encryption: All communication channels established by the Application are strictly point-to-point and encrypted in transit using industry-standard Transport Layer Security (TLS 1.3) protocols directly routed to Google's official API gateways.

6. Compliance with Google API User Data Policy

The Application's use and transfer to any other app of information received from Google APIs will adhere to the Google API User Data Policy, including the Limited Use requirements.

7. Modifications to This Policy

We reserve the right to update this Privacy Policy to align with evolving Google Cloud Platform compliance mandates or advancements in our Model Context Protocol implementations. Any updates will be pushed directly to this official compliance domain.
